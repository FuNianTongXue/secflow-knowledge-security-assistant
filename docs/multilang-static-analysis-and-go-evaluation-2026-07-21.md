# 多语言静态分析与 Go OWASP 基线评估

评估日期：2026-07-21

## 实现范围

扫描链路现支持 Java、Python、Go、C、C++、Rust 与 Solidity：

- Tree-sitter 生成 AST，并提取 import、类型、函数及可审计的节点/父子边预览。
- 在函数范围内生成结构化 CFG 节点、顺序边、条件分支边和循环回边。
- 从声明及赋值表达式生成变量级 DFG 节点和传播边。
- Semgrep OSS 离线 taint 规则补充 source→sink 数据流；结果与 Tree-sitter 控制条件合并。
- Java 继续使用已有跨方法分析器；其他新增语言当前为文件内 CFG/DFG 与 Semgrep Community Edition 可提供的 taint 范围，不宣称编译器级路径完备性。

离线规则目录共 53 条规则，覆盖 Java 以及 Python、Go、C/C++、Rust、Solidity，规则元数据映射 OWASP Top 10 与 CWE。

本次 Go 源码基线直接覆盖 A01、A02、A03、A05、A07、A10，共 6/10 个 OWASP Top 10 2021 类别。A04/A06/A08/A09 仍需要架构设计、依赖/SCA、完整性或运行态日志证据，不能从这次 Semgrep 源码扫描中可靠推断；报告不把未扫描类别计作“通过”。

## 带标签烟测

15 个自建正反样例用于验证多语言规则链路，结果为 TP=7、TN=8、FP=0、FN=0；观测准确率 100%，观测误报率和漏报率均为 0%。详见 [带标签烟测结果](multilang-labeled-smoke-results.json)。随机 598+598 外部样例的独立评测见 [Go 外部语料资格评测](go-external-598x2-qualification-2026-07-22.md)。

这只是功能烟测，不是生产准确率证明。按“零错误时单侧 95% 精确二项上界低于 0.5%”计算，正例和反例至少各需 598 个独立标签样例。当前 7 个正例对应的漏报率上界约为 34.82%，8 个反例对应的误报率上界约为 31.23%，因此不能声称已经达到“准确率 ≥95%、误报率/漏报率 ≤0.5%”的生产验收目标。

## 指标解释

普通开源项目没有逐规则的漏洞 ground truth。100 个真实项目只能衡量扫描完成率、文件覆盖、解析稳定性、告警规模和人工抽查结果，不能计算 TP、FP、TN、FN，也不能据此计算准确率、误报率或漏报率。原始报告因此将这三个字段保留为 `null`。

生产验收需要独立于规则开发的 Go 标注集，至少包含 598 个漏洞正例和 598 个安全反例，并按规则类别、框架、数据流跨度和项目来源分层。评测器 `scripts/score_labeled_security_corpus.py` 已把点估计门槛与样本量资格分开，样本量不足时不会给出生产通过结论。
