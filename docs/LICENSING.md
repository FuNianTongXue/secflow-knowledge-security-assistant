## 许可证性质

本项目采用 **SecFlow Source-Available Commercial Non-Redistribution License 1.0**。源码公开可见，但该许可证不是经 OSI 认可的开源许可证。

公开源码的目的包括代码审阅、学习、安全研究、产品评估与内部非生产测试。生产部署、商业使用、企业评估之外的内部使用、SaaS 运营、客户交付和再分发需要单独的书面商业授权。

## 常见问题

### 可以 Fork 这个仓库吗？

许可证禁止未经书面许可发布修改或未修改副本以及镜像仓库。GitHub 的公开 Fork 会形成可访问副本，因此在 Fork、镜像或发布派生仓库前应先取得书面许可。

### 可以修改代码吗？

可以为私人内部评估进行修改，但不能未经许可公开发布、交付或再分发修改后的源码和二进制。

### 公司内部不收费，是否属于商业使用？

许可证明确区分“内部非生产评估”和“超出评估的企业内部使用”。后者即使不直接收费，也需要书面商业授权。

### 可以把构建后的 macOS 应用发给其他人吗？

不可以默认再分发。项目作者在官方 Release 中发布的试用包不代表第三方获得重新分发该安装包的权利。

### 可以引用少量代码或文档吗？

安全研究和教学参考属于允许场景，但具体复制行为仍可能受到许可证与适用法律限制。需要发布项目代码、较大篇幅内容或衍生作品时，应先取得书面许可。

## 商业授权范围

以下行为需要与作者签订单独的书面商业许可：

- 生产环境部署或持续性企业内部使用
- 商业项目、产品、平台或服务集成
- SaaS、托管服务、代运营或对外 API
- 客户交付、转售、转授权或任何形式的变现分发
- 需要再分发源码或二进制的合作模式

授权咨询可通过作者的 [GitHub 主页](https://github.com/FuNianTongXue) 联系。请说明使用主体、部署方式、预计用户范围、是否对外提供服务以及是否需要再分发。
