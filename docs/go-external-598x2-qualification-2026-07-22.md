## 语料与隔离策略

本次使用两个独立维护的外部来源，而不是由 SecFlow 规则反向生成样例：

- `securego/gosec`，固定 commit `45b083a0cb42119d61b1e0e364d8c62c68f6f5cd`
- `semgrep/semgrep-rules`，固定 commit `647d58eb2e085440618782963ff094178d98d2fe`

按原始规则 CWE 对齐后，去重池包含 849 个漏洞正例和 620 个安全反例。使用固定随机种子 `20260722` 无放回抽取 598 个正例与 598 个反例作为资格集；剩余 251 个正例和 22 个反例作为诊断集。

GoSec 样例按规范化多文件内容 SHA-256 去重；Semgrep 样例按原始路径和目标行去重。匹配时要求外部规则 CWE 与 SecFlow 告警 CWE 相交，避免用不同类型的告警误算为命中。

资格集在规则冻结前不参与调优。最终规则 SHA-256 为：

`6e5c56a5b00528cbd9c9f426f5d480888a4aaa4d2717db6c97e4d95b5a353113`

清单见 [随机资格清单](../config/evaluation/go-external-random-598x2-2026-07-22.json)，逐样例结果见 [资格评测 JSON](go-external-qualification-results.json)。

## 诊断优化

初始 7 条 Go 规则在诊断集上的结果：TP=24、TN=22、FP=0、FN=227，召回率 9.56%。

诊断阶段增加了 CLI/环境输入、常见 Web 参数 source，以及高置信硬编码凭据、TLS 证书验证、SSH host key 和 pprof 暴露检查。泛化的“变量名 + 长字符串”凭据规则虽然提升诊断召回，却在 100 个真实项目上把候选从 31 放大到 1517，因此被收紧为明确 provider token、长十六进制和带填充 Base64 格式。

冻结前最终诊断结果：TP=57、TN=22、FP=0、FN=194，准确率 28.94%，召回率 22.71%。

## 统计限制

1196 个观察值均有唯一外部标签和唯一来源位置或内容哈希，但它们来自两个上游测试套件，同一套件内仍可能共享框架、模板或规则设计，因此不能视为严格 IID 的真实世界漏洞样本。该评测比自建烟测更独立，可用于回归和能力差距分析，但不应包装成生产现场准确率证明。

后续达到目标需要引入 Go SSA/调用图分析、依赖与配置扫描，以及新的第三方盲测集。不能通过在当前资格集上继续调参再复测来宣称达标。
